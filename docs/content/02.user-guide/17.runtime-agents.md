# 运行时 AGENTS 规则

Lime 应用运行时会话不读取仓库根 `AGENTS.md`，而是使用 `.lime/AGENTS.md` 体系。

## 当前事实源

运行时 AGENTS 相关能力当前分成两层：

1. 运行时 AGENTS 主层

- 全局：`~/.lime/AGENTS.md`
- 工作区：`<workspace>/.lime/AGENTS.md`

这两层会直接进入 Lime 运行时 AGENTS 提示拼接。

2. 本机私有补充层

- `<workspace>/.lime/AGENTS.local.md`

这层默认通过记忆来源配置进入当前 workspace 的有效记忆，不会向父目录递归查找。

## 当前加载口径

默认情况下，可以按下面这条顺序理解：

1. 先读全局：`~/.lime/AGENTS.md`
2. 再读工作区：`<workspace>/.lime/AGENTS.md`
3. 如果保留默认记忆来源配置，再把 `<workspace>/.lime/AGENTS.local.md` 作为本机私有补充带入

推荐分工：

- `~/.lime/AGENTS.md`
  - 放长期个人偏好
- `<workspace>/.lime/AGENTS.md`
  - 放团队共享的项目规则
- `<workspace>/.lime/AGENTS.local.md`
  - 放不想提交到仓库的本机补充

## 在哪里生成模板

当前在 `设置 → 记忆` 页面可以显式生成这些模板：

- 全局模板
- Workspace 模板
- Workspace 本机模板

如果是本机模板，还可以继续一键把 `.lime/AGENTS.local.md` 写入当前 Workspace 的 `.gitignore`。

## 新项目提示

首次创建新项目后，Lime 会弹出一个非强制提示，允许你一键初始化：

- `.lime/AGENTS.md`
- `.lime/AGENTS.local.md`

同时自动补 `.gitignore` 的本机模板忽略规则。

## 注意事项

- Lime 运行时不会读取仓库根 `AGENTS.md`
- `.lime/AGENTS.md` 与 `.lime/AGENTS.local.md` 都只看当前 workspace 根目录
- 如果团队要共享规则，请提交 `.lime/AGENTS.md`
- 如果规则只属于你自己，请放进 `.lime/AGENTS.local.md`

## 最小示例

`~/.lime/AGENTS.md`

```md
# 我的全局 Lime 运行时规则

- 默认使用中文简体
- 先给结论，再给关键步骤
- 没必要时保持简洁
```

`<workspace>/.lime/AGENTS.md`

```md
# 当前项目运行时规则

- 先读后写
- 只改当前任务直接相关内容
- 若无法完整验证，需要明确说明
```

`<workspace>/.lime/AGENTS.local.md`

```md
# 本机私有补充

- 优先使用本机已安装工具链
- 涉及大体量编译时先跑定向测试
```
