# 品牌人设知识库质量检查表

生成完成后逐项检查。

## 事实完整性

- 是否说明资料类型 `brand-persona`、适用场景和使用边界？
- 是否区分事实、品牌主张、推断和待补充信息？
- 是否避免编造来源资料没有提供的品牌荣誉、市场地位、用户评价、案例或承诺？
- 是否把不确定内容标注为 `待补充`？

## 结构完整性

- 是否覆盖品牌内核、受众关系、声音语气、表达模式、标志性素材、禁忌红线和危机升级？
- 是否包含可运行时调用的“智能体应用指南”？
- 是否包含禁止表达和人工升级条件？
- 是否适合后续持续维护？

## 可调用性

- Agent 是否能用它生成品牌故事、内容文案、客服话术或危机回应草稿？
- 是否有清晰的语气示例、渠道差异和禁用表达？
- 是否说明了“不应该怎么写 / 什么必须先确认”？

## Runtime 安全边界

- 是否明确把输出作为 persona 资料，而不是运行时指令？
- 是否只引用来源材料中出现的事实？
- 是否把冲突定位、语气漂移、敏感信息和合规风险写入 warnings 或待确认清单？
- 是否能写回 Agent Knowledge `documents/<pack-name>.md`，并适配 `profile: document-first` 与 `runtime.mode: persona`？
