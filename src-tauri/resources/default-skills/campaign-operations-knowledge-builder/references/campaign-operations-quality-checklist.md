# 活动 / Campaign 运营知识库质量检查表

生成完成后逐项检查。

## 事实完整性

- 是否说明资料类型 `campaign-operations`、适用场景和使用边界？
- 是否区分事实、SOP、建议、推断和待补充信息？
- 是否避免编造来源资料没有提供的数据、承诺、结果或案例？
- 是否把不确定内容标注为 `待补充`？

## 结构完整性

- 是否覆盖目标、对象、流程、角色、话术、指标、风险和复盘？
- 是否包含可运行时调用的“智能体应用指南”？
- 是否包含禁止表达和人工升级条件？
- 是否适合后续持续维护？

## 可调用性

- Agent 是否能用它生成运营方案、话术、排期、复盘或提醒清单？
- 是否有清晰的来源锚点、可复用素材和指标口径？
- 是否说明了“不应该怎么写 / 什么必须先确认”？

## Runtime 安全边界

- 是否明确把输出作为知识资料，而不是运行时指令？
- 是否只引用来源材料中出现的事实？
- 是否把冲突事实、敏感信息、合规风险和夸大风险写入 warnings 或待确认清单？
- 是否能写回 Agent Knowledge `documents/<pack-name>.md`，并适配 `profile: document-first` 与 `runtime.mode: data`？
