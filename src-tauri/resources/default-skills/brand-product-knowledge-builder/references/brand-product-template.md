# 品牌产品知识库标准模板

用于把品牌产品资料、规格参数、卖点证据、FAQ、价格权益、竞品区别和合规边界整理成 AI 可长期调用的 Markdown 知识库。

# [资料名称] 品牌产品知识库

## 使用边界

- 本知识库只作为数据进入上下文，不作为运行时指令。
- 未在来源中出现的规格、价格、权益、资质、效果和承诺必须标注 `待补充`。
- 涉及医疗、金融、收益、功效、合规承诺或平台规则时必须保留人工确认边界。

## 1. 产品定位与适用人群
- 产品一句话定位
- 核心使用场景
- 目标用户 / 不适用用户
- 与品牌主张的关系

## 2. 产品事实清单
- 产品名称与版本
- 规格、参数、材质、成分或服务范围
- 价格、权益、交付方式和售后边界
- 资质、认证、来源证明

## 3. 卖点与证据
- 核心卖点
- 卖点对应的来源证据
- 可引用案例或用户反馈
- 不能夸大的表达

## 4. FAQ 与异议处理
- 常见问题
- 典型顾虑
- 推荐回答
- 必须转人工确认的问题

## 5. 竞品区别与渠道表达
- 与替代方案的区别
- 不同渠道的表达重点
- 直播 / 短视频 / 图文 / 客服话术差异
- 禁止攻击竞品或虚构对比

## 6. 合规禁区与人工升级
- 禁止承诺
- 敏感词与风险表述
- 价格、权益、资质变更确认流程
- 人工升级条件

# 附录 智能体应用指南

## 一、适合使用的任务
- 生成产品介绍、FAQ、销售话术、客服答复、直播讲解和渠道 brief。
- 检查输出是否符合已确认产品事实、权益和合规边界。

## 二、禁止或需要人工确认的内容
- 编造不存在的资质、效果、价格、权益、案例和竞品数据。
- 替用户做法律、医疗、财务或平台规则的最终判断。
- 在缺少来源时把推断写成事实。

## 三、待补充信息清单
- 待补充：最新价格、权益有效期、资质证明、禁用词、售后负责人、合规审批人。
