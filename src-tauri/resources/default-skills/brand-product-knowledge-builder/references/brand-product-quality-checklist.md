# 品牌产品知识库质量检查表

生成完成后逐项检查。

## 事实完整性

- 是否说明资料类型 `brand-product`、适用场景和使用边界？
- 是否区分事实、卖点、证据、建议、推断和待补充信息？
- 是否避免编造来源资料没有提供的规格、价格、权益、资质、效果或案例？
- 是否把不确定内容标注为 `待补充`？

## 结构完整性

- 是否覆盖定位、人群、事实清单、卖点证据、FAQ、竞品区别、渠道表达和合规禁区？
- 是否包含可运行时调用的“智能体应用指南”？
- 是否包含禁止表达和人工升级条件？
- 是否适合后续持续维护？

## 可调用性

- Agent 是否能用它生成产品介绍、FAQ、客服答复、销售话术或直播讲解？
- 是否有清晰的来源锚点、权益口径、资质口径和渠道差异？
- 是否说明了“不应该怎么写 / 什么必须先确认”？

## Runtime 安全边界

- 是否明确把输出作为知识资料，而不是运行时指令？
- 是否只引用来源材料中出现的事实？
- 是否把冲突事实、敏感信息、合规风险和夸大风险写入 warnings 或待确认清单？
- 是否能写回 Agent Knowledge `documents/<pack-name>.md`，并适配 `profile: document-first` 与 `runtime.mode: data`？
