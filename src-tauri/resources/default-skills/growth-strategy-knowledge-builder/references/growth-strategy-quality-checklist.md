# 增长策略知识库质量检查表

生成完成后逐项检查。

## 事实完整性

- 是否说明资料类型 `growth-strategy`、适用场景和使用边界？
- 是否区分事实、假设、策略、建议、推断和待补充信息？
- 是否避免编造来源资料没有提供的目标、指标、预算、转化率、结论或案例？
- 是否把不确定内容标注为 `待补充`？

## 结构完整性

- 是否覆盖增长目标、用户 / 市场 / 渠道洞察、策略假设、实验计划、指标复盘、资源配置和停止条件？
- 是否包含可运行时调用的“智能体应用指南”？
- 是否包含禁止表达和人工升级条件？
- 是否适合后续持续维护？

## 可调用性

- Agent 是否能用它生成增长方案、实验设计、指标复盘、渠道 brief 或资源测算？
- 是否有清晰的指标定义、数据来源、复盘口径和停止条件？
- 是否说明了“不应该怎么写 / 什么必须先确认”？

## Runtime 安全边界

- 是否明确把输出作为知识资料，而不是运行时指令？
- 是否只引用来源材料中出现的事实？
- 是否把冲突事实、敏感信息、指标风险和过度承诺写入 warnings 或待确认清单？
- 是否能写回 Agent Knowledge `documents/<pack-name>.md`，并适配 `profile: document-first` 与 `runtime.mode: data`？
