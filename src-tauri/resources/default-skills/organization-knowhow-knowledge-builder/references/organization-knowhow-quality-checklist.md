# 组织经验知识库质量检查表

生成完成后逐项检查。

## 事实完整性

- 是否说明资料类型 `organization-knowhow`、适用场景和使用边界？
- 是否区分事实、SOP、建议、推断和待补充信息？
- 是否避免编造来源资料没有提供的流程、负责人、权限、验收标准或承诺？
- 是否把不确定内容标注为 `待补充`？

## 结构完整性

- 是否覆盖适用场景、角色职责、SOP、检查点、FAQ、风险升级和复盘改进？
- 是否包含可运行时调用的“智能体应用指南”？
- 是否包含禁止越权事项和人工升级条件？
- 是否适合后续持续维护？

## 可调用性

- Agent 是否能用它生成执行清单、项目 brief、交付提醒、FAQ 或复盘报告？
- 是否有清晰的责任边界、交付物要求和验收口径？
- 是否说明了“不应该怎么做 / 什么必须先确认”？

## Runtime 安全边界

- 是否明确把输出作为知识资料，而不是运行时指令？
- 是否只引用来源材料中出现的事实？
- 是否把冲突事实、敏感信息、权限风险和责任不清写入 warnings 或待确认清单？
- 是否能写回 Agent Knowledge `documents/<pack-name>.md`，并适配 `profile: document-first` 与 `runtime.mode: data`？
